Задание для лабораторной работы 1: написать программу на языке C/C++ для перемножения двух квадратных матриц. Исходные данные: файл(ы), содержащий значения исходных матриц. Выходные данные: файл со значениями результирующей матрицы, время выполнения, объем задачи. Обязательна автоматизированная проверка результатов вычислений с помощью сторонних библиотек или программного обеспечения (например, на Matlab/Python).

1.	Разработка программы на C++
Создана программа для умножения квадратных матриц с использованием классического алгоритма (три вложенных цикла). Исходные матрицы считываются из файлов, результат сохраняется в файл. Время измеряется функцией clock().
2.	Генерация тестовых данных
Скриптом на Python сгенерированы случайные матрицы размером 100, 200, 400, 800, 1200 и 1600 элементов.

Выводы
В ходе выполнения лабораторной работы была достигнута поставленная цель — разработана программа на C++ для умножения квадратных матриц с автоматизированной верификацией результатов.
1. Корректность реализации
Сравнение результатов, полученных программой на C++, с эталонным умножением в Python с использованием библиотеки NumPy подтвердило правильность работы программы. Для всех исследованных размеров матриц (100, 200, 400, 800, 1200, 1600) результаты совпали с высокой точностью. Это доказывает, что алгоритм умножения реализован верно и программа может использоваться для вычислений.
2. Анализ производительности
Полученные экспериментальные данные подтверждают теоретическую сложность алгоритма умножения матриц O(N³). При увеличении размера матрицы в 2 раза время выполнения возрастает примерно в 8 раз:
•	с 100 до 200: время увеличилось с 0.008 до 0.066 сек (в 8.25 раза)
•	с 200 до 400: с 0.066 до 0.751 сек (в 11.4 раза)
•	с 400 до 800: с 0.751 до 7.267 сек (в 9.7 раза)
•	с 800 до 1600: с 7.267 до 119.912 сек (в 16.5 раза)
Рост времени для больших матриц превышает теоретический из-за увеличения нагрузки на кэш-память процессора и ограничений пропускной способности памяти.
3. Практическая применимость
Разработанная программа может использоваться для умножения матриц различных размеров. Для матриц размером до 800×800 время выполнения не превышает 10 секунд, что удобно для интерактивной работы. Для матриц 1200×1200 время составляет около 50 секунд, для 1600×1600 — около 2 минут, что приемлемо для пакетной обработки данных.
